## How I think about AI engineering

**Evaluation before deployment.** Most AI failures aren't model failures — they're untested edge cases that ship anyway. I design benchmarking frameworks that stress-test for accuracy, consistency, and adversarial inputs *before* a model goes near production.

**Retrieval over hallucination.** If a system needs to be right, it needs to know what it doesn't know. I build RAG and agent architectures that ground generation in real, retrievable context rather than betting on parametric memory.

**Production over prototype.** A computer vision model that works in a notebook and one that survives occlusion, motion blur, and identity loss in a live feed are different engineering problems. I solve for the second one.
